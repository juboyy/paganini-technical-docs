Deterministic AIOS Fabric

O Paganini AIOS opera como um tecido de máquinas determinísticas, onde cada interação é um evento rastreável, transformando intenções do usuário em ações auditáveis e retroalimentando o sistema para auto-aprimoramento contínuo.

Fluxo Principal​

Propriedades do Fabric​

Determinismo​

Cada input produz um caminho de execução rastreável. O pipeline BMAD-CE garante que não existem "atalhos" — toda ação passa por gates documentados.

Auditabilidade​

Todos os artefatos são persistidos:

  • Gate tokens em cada commit
  • Pipeline logs em runtime/pipeline-log.jsonl
  • Decision logs em memory/gate-log.jsonl
  • Daily logs em memory/YYYY-MM-DD.md

Auto-Aprimoramento​

O Stage 19 (Cognitive Self-Audit) monitora drift entre o comportamento atual e o TASTE.md/SOUL.md. Violações são registradas em memory/violations.md.

Modelo de Memória​

O sistema opera com 3 camadas de memória simultâneas:

LayerEngineDimensõesPropósito
1Filesystem (Markdown)N/AMemória editável por humanos, versionada em Git
2pgvector (Supabase EC2)3072dBusca semântica de alta precisão, particionamento por agente
3Mem0 (Qdrant local)768dLayer aditiva, associativa, rápida
Nota importante

As dimensões dos vetores são incompatíveis entre layers (3072d vs 768d). Cada layer serve um propósito distinto e não são intercambiáveis.
