Protocolo Agent-to-Agent (A2A)

O Paganini AIOS não é monolítico. Ele orquestra múltiplos agentes especializados através de um protocolo de comunicação estruturado.

Fluxo de Interação​

Webhook de Comunicação​

PropriedadeValor
Endpointhttp://localhost:<port>/webhooks/oracli-task
AutenticaçãoHMAC-SHA256 com secret <redacted>
FormatoJSON com task description + context package
TimeoutConfigurável por task tier

DimensãoValor

Learning loopSkills se auto-aprimoram com uso
PresençaTelegram (João pode falar diretamente)

Regras do Protocolo​

  1. Context package obrigatório — COSMO nunca recebe task "bare" sem contexto rico
  2. Chunking — Máximo 3-5 items por dispatch
  3. Escalation ≠ failure — É sinal de que o sistema funciona
  4. Results always written — Mesmo em falha, COSMO escreve resultado com diagnóstico
  5. Knowledge persists — Stage 17 executa em ambos os lados (COSMO persiste, OraCLI curate)
