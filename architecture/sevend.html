Iron Laws (5 Regras Invioláveis)​

Do REASONING.md — aplicam-se a todos os níveis:

#Iron LawConsequência da Violação
1No code without design firstStop, revert, brainstorm
2No production code without failing test first (TDD)Stop, write test, then code
3No fixes without root cause investigationStop, investigate, then fix
4No completion claims without fresh verificationEvidence > claims
5No implementation without gate tokenStop, run gate.py, proceed
Violações Documentadas

12 violações registradas desde fevereiro 2026, todas em memory/violations.md. O pattern mais comum: implementação inline sem gate token (behavioral, não documental — corrigido com gate.py como ação concreta).
