Cyber-Security & Hardening (Level 3)

Sandbox Isolation​

Isolamento total via Docker Containers por Tenant. O agente opera em ambiente Bubblewrap/Seatbelt com regras de negação nativas:

Regras de Negação​

RegraDescrição
~/.ssh bloqueadoProibição de acesso a chaves SSH
~/.aws bloqueadoProibição de acesso a credenciais AWS
.env global bloqueadoBloqueio de leitura de variáveis de ambiente globais
Egress filteringBloqueio de curl/wget para destinos não-autorizados
NetworkApenas HTTPS (porta 443). TCP direto bloqueado

Limitações Conhecidas (Documentadas)​

LimitaçãoImpactoWorkaround
TCP direto bloqueadoPostgres (5432/6543) inacessívelSQL via Supabase REST API ou CLI externo
IPv6 blockedAlgumas conexões falhamForçar IPv4
/dev/shm limitadoChromium pode falharFlag --disable-dev-shm-usage

Rede Confiável​

Multi-Tenant Architecture​

Decisão arquitetural: Container por Tenant (NOT shared pool). Dados financeiros exigem isolamento total.

AspectoDecisão
IsolamentoContainer dedicado per tenant
DatabaseDatabase isolado per tenant
WorkspaceWorkspace isolado per tenant
SecretsSecrets isolados per tenant
BaseK8s (Helm chart existente)
LifecycleNamespace + Deployment per tenant
Estimativa6-10 semanas para MVP multi-tenant
