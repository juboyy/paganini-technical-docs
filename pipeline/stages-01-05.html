Stages 01-05: Planejamento & Design

Stage 01: Context Scout & Logic Check​

STAGE 01 — OWNER: OraCLIContext Scout & Logic Check

Mapeamento detalhado do codebase via GitNexus, consulta a memórias passadas (pgvector), e verificação de conflitos de decisão com Decision Conflict Monitor. Esta é a porta de entrada para qualquer tarefa.

Ferramentas: GitNexus (análise de impacto), pgvector (memória RAG), KuzuDB (ontologia), Scripts de Validação
Artefatos: Context Package, Relatório de Conflitos de Decisão (se houver)

# Exemplo real de Context Scout
python3 scripts/run_pipeline.py "implementar novo endpoint" --tier quick --dry-run
# Output: gate token + execution plan + blast radius

Stage 02: Product Owner Definition​

STAGE 02 — OWNER: COSMOProduct Owner Definition

Transformação da requisição inicial em um Product Requirements Document (PRD) formal, com escopo claro, objetivos, KPIs e critérios de aceitação.

Ferramentas: Gemini (geração de PRD), Memória (referências de produto), Linear (vinculação ao roadmap)
Artefatos: prd.md

# Estrutura típica de PRD
title: "Feature X"
objective: "..."
kpis:
  - metric: "Latência P99"
    target: "&lt;200ms"
acceptance_criteria:
  - "Testes E2E passando"
  - "Code review aprovado"
  - "Zero breaking changes"

Stage 03: UX/UI & Prototyping​

STAGE 03 — OWNER: COSMOUX/UI & Prototyping

Criação de wireframes, mockups e protótipos de interface. Usa DesignKit para extração de design system e Frontend-Design para geração de código UI.

Ferramentas: PinchTab (navegação), DesignKit (design system), Frontend-Design (UI generation)
Artefatos: Wireframes, Protótipos, Specs de UI/UX

Design Direction: Editorial premium ("Bloomberg Terminal meets luxury magazine"), dark-first, Instrument Serif + DM Sans + JetBrains Mono, navy+gold.

Stage 04: Architecture Definition​

STAGE 04 — OWNER: COSMOArchitecture Definition

Definição da arquitetura de software e infraestrutura. Inclui a escolha de tecnologias, padrões de design e desenho de componentes.

Ferramentas: Visual Explainer (diagramas), GitNexus (análise de codebase), Memória (decisões passadas)
Artefatos: architecture.md, Diagramas de Componentes/Fluxos

Stage 05: Data Model & Schema​

STAGE 05 — OWNER: COSMOData Model & Schema

Design do modelo de dados, schema de banco de dados e APIs (GraphQL/REST). Garante integridade e eficiência do armazenamento.

Ferramentas: Supabase (gerenciamento de schema), Gemini (design de schema)
Artefatos: Schema SQL/migrations, Specs de API

-- Exemplo: tabela de capabilities (real, Supabase apkflemxmsbdltziouls)
CREATE TABLE capabilities (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  kind TEXT CHECK (kind IN ('integration','tool','skill','native','script','api')),
  description TEXT,
  agents TEXT[],
  embedding VECTOR(3072),
  status TEXT DEFAULT 'active',
  created_at TIMESTAMPTZ DEFAULT now()
);
-- 34 entries, 6 kinds, semantic search via gemini-embedding-001
