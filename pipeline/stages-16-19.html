Stages 16-19: Deploy, Conhecimento & Auto-Melhoria

Stage 16: Production Deploy​

STAGE 16 — OWNER: COSMO (após aprovação Linear)Production Deploy

Após aprovação humana no Linear, a funcionalidade é mergida e deployada para produção. Inclui smoke tests pós-deploy.

Ferramentas: Vercel (--prod), GitHub Actions, Scripts de Smoke Test
Artefatos: Log de Deploy, Relatório de Smoke Test

# Post-approval pipeline (triggered automatically)
# 1. Playwright E2E
# 2. PR validation (lint/types/tests)
# 3. Security scan (npm audit, secrets)
# 4. Demo video
# 5. Merge
# 6. Deploy prod
npx vercel deploy --prod --token "$VERCEL_TOKEN" --yes
# 7. Smoke test
# 8. Update all systems (Jira/Linear/Confluence/Slack/Memory/ROI)

Stage 17: Knowledge Persistence​

STAGE 17 — OWNER: COSMO escreve, OraCLI curate — 🔴 OBRIGATÓRIOKnowledge Persistence

Captura e organização do conhecimento gerado. Cada tarefa ensina a próxima. Este stage é obrigatório sem exceções.

Ferramentas: Memória (MEMORY.md, daily logs), Confluence, Git
Artefatos: MEMORY.md atualizado, daily-log.md, Documentação Técnica

O que é persistido:

  • Decisões tomadas e razões
  • Erros encontrados e soluções
  • Padrões descobertos
  • Conflitos com decisões anteriores
  • Métricas de performance

Stage 19: Cognitive Self-Audit​

STAGE 19 — OWNER: Automático (Cron) — 🧠 AGENTIC DRIFT PREVENTIONCognitive Self-Audit

Job periódico e autônomo que audita o codebase e o comportamento do agente contra TASTE.md (padrões de qualidade) e SOUL.md (diretrizes). Detecta e previne "Agentic Drift".

Ferramentas: Cron Jobs, Scripts de Auditoria (Python), Gemini (análise de conformidade)
Artefatos: Relatório de Auditoria Cognitiva, Propostas de Refatoração

Checklist de auditoria (executado no heartbeat):

  • Código escrito inline sem pipeline classification?
  • gate.py rodado antes de cada implementação?
  • Code Agent + Codex usado para implementação?
  • Linear issues criados ANTES da implementação?
  • Context Scout (Stage 1) executado antes de cada tarefa?
  • Stage 17 (Knowledge) completado após cada tarefa?

Se violações encontradas: append imediato a memory/violations.md.
