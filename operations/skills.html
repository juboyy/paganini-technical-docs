Princípios de Design de Skills​

MECE (Mutually Exclusive, Collectively Exhaustive)​

  • Cada tipo de trabalho tem uma e apenas uma skill
  • Sem sobreposição de responsabilidades
  • Se duas skills cobrem o mesmo domínio → merge ou delete

Ciclo de Vida​

Durable Agent Rule​

"Se precisou ser pedido duas vezes, a automação falhou."

Toda task solicitada uma vez deve virar:

  1. Skill (SKILL.md + scripts)
  2. Cron (agendamento automático)
  3. State file (rastreamento de estado)

Capabilities Graph (Supabase)​

Todas as 34+ capabilities estão indexadas na tabela capabilities do Supabase com embeddings semânticos (3072d):

-- Busca semântica de capabilities
SELECT * FROM search_capabilities(<query_embedding>, 5);

-- Filtrar por tipo
SELECT * FROM capabilities WHERE kind = 'skill' AND status = 'active';

-- Filtrar por agente
SELECT * FROM capabilities WHERE agents @> '{oracli}' AND status = 'active';
